# MAIA Engine — Web 用户交互界面设计

> 探索式多智能体交互界面 · 设计文档 v0.1

---

## 一、设计目标

围绕「探索即扩展」的核心理念，设计一个以**工作区间**为中心、以**自然语言对话**为主交互通道的 Web 界面。用户在界面中探索数据、发出需求、核验自动生成的代码、编排多智能体流程、查看处理结果——所有操作由交互 Agent 统一引导。

### 设计原则

| 原则 | 说明 |
|------|------|
| **对话驱动** | 交互 Agent 是界面的"灵魂"，用户通过对话完成大部分操作 |
| **渐进式披露** | 默认简洁，按需展开复杂功能（编排编辑器、代码核验等） |
| **工作区间中心化** | 所有操作围绕当前工作区间展开，上下文始终可见 |
| **即时反馈** | 任务提交后实时展示执行进度、中间结果、日志流 |
| **可逆可改** | 编排文件、生成代码均可人工修改后再提交 |
| **空间可浏览** | 工具空间、数据空间等资源可浏览、搜索、预览 |

## 三、视图模式

中央主工作区根据用户当前操作，在不同视图间切换。视图切换由交互 Agent 驱动或用户手动切换。

### 3.1 视图类型

```
视图模式状态机:

   [对话视图] ◄──── 默认入口
       │
       ├── 用户发出需求 ──→ [对话视图] (Agent 回复，内联结果卡片)
       │
       ├── 需要代码核验 ──→ [代码核验视图] (并排: 代码 + 沙箱预览)
       │
       ├── 复杂编排任务 ──→ [编排编辑器视图] (Markdown 编排文件)
       │
       ├── 查看数据详情 ──→ [数据预览视图] (表格/图像/波形等)
       │
       └── 查看执行过程 ──→ [任务监控视图] (DAG 图 + 日志流)
```

### 3.2 视图切换规则

- **自动切换**: 交互 Agent 根据上下文自动切换视图（如生成编排文件后自动跳转编排编辑器）
- **手动切换**: 顶部 Tab 栏始终可见，用户可手动切换
- **返回对话**: 所有视图顶部保留"返回对话"入口，对话不丢失

交互行为：
- **单击文件**: 右侧属性面板展示元数据 + 主工作区预览内容

- **多选**: Ctrl/Cmd + 点击多选，作为批量输入
- **上传**: 拖拽文件到树中任意位置，自动识别格式并注册到数据空间

对话视图中的**内联卡片类型**：

| 卡片类型 | 触发场景 | 交互 |
|---------|---------|------|
| 数据预览卡片 | Agent 解析了用户选择的数据 | 展开预览、查看 Schema、修改输入 |
| 工具匹配卡片 | 资源发现器返回匹配结果 | 直接执行 / 查看详情 / 触发代码生成 |
| 代码核验卡片 | 资源构建器生成了新代码 | 打开核验视图 / 快速批准 / 快速拒绝 |
| 执行进度卡片 | 任务开始执行 | 实时进度条、展开看详细日志 |
| 结果摘要卡片 | 任务执行完成 | 预览结果、下载、注册到数据空间 |
| 编排预览卡片 | 生成了编排描述文件 | 打开编排编辑器 / 快速确认执行 |

核验流程三步走：
1. **自动检查**: 语法、依赖、沙箱预跑——全部通过后才呈现给用户
2. **人工审查**: 用户查看代码逻辑、沙箱输出、修改工具描述
3. **决策**: 批准（注册到工具空间）/ 修改后批准 / 拒绝（丢弃）

- **Markdown 文本编辑** (右侧): 用户可直接修改编排描述文本

- **双向同步**: 修改 Markdown → DAG 图自动更新；拖拽 DAG → Markdown 自动更新

支持的 DAG 编辑操作：
- 拖拽调整节点位置
- 拖拽连线改变依赖关系
- 双击节点编辑 Agent 参数
- 右键节点: 替换 Agent、删除、复制
- 添加条件分支节点
- 设置并行/串行策略

## 五、核心交互流程

### 5.1 简化模式：数据处理需求

```
用户在左侧面板选择数据
    │
    ▼
在对话区输入需求 "帮我分析这批EEG数据中的异常信号"
    │
    ▼
交互 Agent 解析意图 → 查询工具空间
    │
    ├── 找到工具 ──→ 展示工具卡片 → 用户确认 → 执行 → 结果卡片
    │
    └── 未找到 ──→ Agent 提示"需要生成新工具" → 用户点击"自动生成代码"
            │
            ▼
        跳转到代码核验视图
            │
            ├── 通过 ──→ 工具注册 + 执行 → 结果
            └── 修改 ──→ 用户编辑代码 → 再核验 → 通过 → 注册 + 执行
```

### 5.2 复杂模式：多智能体编排

```
用户在对话区用自然语言描述复杂任务
    │
    ▼
交互 Agent → 意图解析 → 流程编译 → 查询资源 → 生成编排 Markdown
    │
    ▼
对话区展示编排预览卡片 → 用户点击"在编排编辑器中打开"
    │
    ▼
编排编辑器视图 — 用户交互修改:
  - 调整 DAG 结构（拖拽）
  - 替换 Agent
  - 修改参数
  - 增加/删除节点
    │
    ▼
用户点击"提交执行"
    │
    ▼
自动切换到任务监控视图 — 实时追踪
    │
    ▼
执行完毕 → 结果卡片在对话中 → 结果注册到数据空间
```

### 5.3 探索式工具增长循环

```
第1次需求: "对EEG做带通滤波"
  → 无工具 → 自动生成 bandpass_filter.py → 核验通过 → 注册 → 执行

第2次需求: "对EEG做带通滤波然后检测异常"
  → 发现 bandpass_filter ✓, 无异常检测 → 生成 anomaly_detector.py → 注册

第N次需求: "对EEG做带通滤波然后检测异常"
  → 两个工具都已有 ✅ → 直接编排执行 — 无需生成代码
  → 工具空间从 0 → 1 → 2 → ... → N 持续增长
```

## 七、关键交互细节

### 7.1 拖拽交互

| 拖拽操作 | 效果 |
|---------|------|
| 文件树 → 对话输入区 | 将文件作为任务输入附件 |
| 资源浏览器中的Agent → 对话区 | 在需求中指定使用该Agent |
| 资源浏览器中的工具 → 对话区 | 在需求中指定使用该工具 |
| 对话区附件 → 文件树 | 将结果保存到指定位置 |
| DAG 节点拖拽 | 调整编排流程结构 |
| 文件/文件夹 → 工作区间 | 上传新数据 |

### 7.2 键盘快捷键

| 快捷键 | 操作 |
|--------|------|
| `Ctrl+K` | 聚焦搜索（命令面板） |

| `Ctrl+B` | 切换左侧面板 |
| `Ctrl+J` | 切换右侧面板 |
| `Ctrl+N` | 新建工作区间 |
| `Ctrl+O` | 上传文件 |
| `Esc` | 关闭当前视图 / 返回对话 |

### 7.3 通知系统

- **Toast**: 右上角短暂弹出（工具注册成功、任务完成）
- **对话内通知**: Agent 在对话流中直接推送状态更新
- **Badge**: 左侧面板任务历史上的未读标记

## 八、技术选型建议

| 层面 | 推荐方案 | 理由 |
|------|---------|------|
| 前端框架 | React + TypeScript | 生态丰富，组件化适合复杂交互 |
| UI 组件库 | 自定义 + Radix UI | 需要高度定制，Radix 提供无样式可访问组件 |
| 代码编辑器 | Monaco Editor | VS Code 同款，支持语法高亮、Diff |
| DAG 可视化 | React Flow | 节点/连线编辑，高度可定制 |
| 数据可视化 | D3.js + Observable Plot | 灵活支持多种数据类型的图表 |
| 实时通信 | WebSocket (Agent ↔ 前端) | 任务进度推送、日志流 |
| 状态管理 | Zustand | 轻量，适合中等复杂度状态 |
| 文件树 | 自研（基于树形数据结构） | 需要拖拽上传、右键菜单、多选 |
| Markdown 编辑 | Milkdown / TipTap | 所见即所得，支持自定义节点 |
