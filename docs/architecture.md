# MAIA Engine 架构设计

> 多智能体驱动的多模态智能处理引擎

---

## 一、核心理念

构建一个多智能体驱动的多模态智能处理引擎，实现对开放世界多源异构数据的智能处理流程的自动化构建与动态扩展，支持在分布式异构算力集群上完成多智能体的协同工作。

## 二、核心设计理念

### 2.1 探索式用户交互

用户在工作区间中选择数据后发出需求，系统自动触发任务编排。如发现没有工具可完成，则自动编写代码，经用户核验后落地为本地工具。后续类似需求直接调用本地工具执行，工具空间随用户探索不断增长。

### 2.2 自然语言驱动的多智能体编排

用户通过自然语言描述复杂任务流程，系统根据已有Agent自动生成编排描述文件（Markdown格式）。用户可交互修改该流程定义，最终形成任务编排描述文件，驱动多智能体协同工作。

### 2.3 一切以Agent为主体

所有处理的最终执行主体都是Agent。包括与用户交互、接收用户探索指令在内，也由一个专门的交互Agent负责。系统本质上是多智能体协作生态系统。

### 2.4 资源全生命周期管理

资源层内所有资源（数据、工具、模型、任务等）均需进行标准化描述、注册登记，支持查询获取和动态整合。所有资源的构建、生成、注册、版本管理及全生命周期管理均由核心层统一把关和支撑。

### 2.5 AI模型作为特殊工具

所有AI模型对应其输出和使用方式，作为特殊工具纳入工具空间管理，同时在模型空间中进行独立的版本管理。模型与工具建立关联关系，便于追踪和调度。

## 三、系统分层

```
┌─────────────────────────────────────────────────────┐
│  应用层 (app/)                                       │
│  探索式交互界面、工作区间、AI助手、任务编排、可视化     │
├─────────────────────────────────────────────────────┤
│  核心层 (core/)                                      │
│  唯一具有执行能力的层次                               │
│  ┌──────────┬──────────┬──────────┬──────────┐      │
│  │ 工作流引擎 │ 资源管理  │ 调度算力  │ 多Agent协作│      │
│  ├──────────┼──────────┼──────────┼──────────┤      │
│  │ 安全治理  │ 可观测性  │          │          │      │
│  └──────────┴──────────┴──────────┴──────────┘      │
├─────────────────────────────────────────────────────┤
│  资源层 (resources/)                                 │
│  数据空间 │ 工具空间 │ 模型空间 │ Agent空间 │ 用户空间 │ 任务空间 │
├─────────────────────────────────────────────────────┤
│  存储层 (storage/)                                   │
│  数据库 │ 文件存储 │ 缓存                             │
└─────────────────────────────────────────────────────┘
```

## 四、核心层六大子系统

### 4.1 工作流引擎子系统 (`core/engine/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 意图解析器 | `intent_parser.py` | 将用户自然语言意图解析为标准化流程描述 |
| 流程编译器 | `flow_compiler.py` | 将流程描述编译为可执行的 DAG |
| 执行器 | `executor.py` | 按 DAG 驱动多智能体执行，管理任务生命周期 |
| 状态管理器 | `state_manager.py` | 维护所有任务实例的执行状态 |
| 观测器 | `observer.py` | 实时追踪流程执行进度，反馈给用户 |

### 4.2 资源生命周期管理子系统 (`core/resource/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 资源构建器 | `builder.py` | 根据用户需求自动编写代码，构建新工具 |
| 资源注册中心 | `registry.py` | 统一管理所有资源的注册、描述、版本 |
| 资源发现器 | `discoverer.py` | 按能力、类型、标签等维度查询和获取资源 |
| 资源生命周期管理器 | `lifecycle.py` | 管理资源的创建、版本升级、废弃、归档全生命周期 |

### 4.3 调度与算力管理子系统 (`core/scheduler/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 异构调度器 | `heterogeneous.py` | 将任务分发至异构算力集群（CPU/GPU/NPU） |
| 资源分配器 | `allocator.py` | 动态分配计算资源（CPU核心数、GPU显存、内存大小） |
| 负载均衡器 | `balancer.py` | 均衡集群中各节点的负载 |

### 4.4 多智能体协作子系统 (`core/agent/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 通信总线 | `bus.py` | 提供 Pub/Sub 和 RPC 两种通信模式 |
| 协作协议 | `protocol.py` | 定义智能体间的交互规范 |
| 角色协调器 | `coordinator.py` | 根据智能体的角色和能力进行任务分配和冲突解决 |
| Agent工厂 | `factory.py` | 负责Agent的实例化、池化管理和生命周期维护 |

### 4.5 安全与治理子系统 (`core/security/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 沙箱引擎 | `sandbox.py` | 隔离执行智能体和工具的代码 |
| 代码核验器 | `verifier.py` | 自动生成代码后提交用户核验 |
| 看门狗 | `watchdog.py` | 监控异常行为，实现故障检测和自动恢复 |
| 审计日志 | `audit.py` | 记录所有操作，支持合规审查 |

### 4.6 可观测性子系统 (`core/observability/`)

| 组件 | 文件 | 职责 |
|------|------|------|
| 监控 | `monitor.py` | 采集系统指标（资源使用率、任务吞吐量、响应延迟） |
| 告警 | `alert.py` | 异常时触发通知 |
| 链路追踪 | `trace.py` | 记录完整调用链，支持分布式追踪 |

## 五、资源层六空间

### 5.1 数据空间 (`resources/data/`)

- **管理内容**: 多源异构数据（文本/图像/视频/EEG/fMRI/传感器等）的元数据及存储位置，以及 Memory 和 Result
- **记录信息**: 数据格式、Schema、版本、存储路径、血缘关系、生成时间、访问权限
- **资源描述**: 标准化描述（名称、类型、来源、格式、时间范围、质量评分等）

### 5.2 工具空间 (`resources/tools/`)

- **管理内容**: 数据处理工具、算法库、存储读写工具、特征提取工具、代码生成工具等（含用户探索生成的本地工具）
- **记录信息**: 工具名称、类型、调用接口定义、版本、代码存储位置、创建时间、使用次数
- **特殊规则**: 用户探索生成的本地工具经核验后注册入库

### 5.3 模型空间 (`resources/models/`)

- **管理内容**: 多模态模型（LLM/ViT/3D-CNN/时序模型等）
- **记录信息**: 模型名称、框架类型、输入输出格式、权重文件路径、版本、部署状态、关联工具ID
- **关联关系**: 每个AI模型对应工具空间中的调用工具

### 5.4 智能体空间 (`resources/agents/`)

- **管理内容**: 多智能体定义，包括交互Agent、任务Agent、编排Agent等
- **记录信息**: 唯一标识、能力描述（Markdown格式）、角色类型、依赖工具列表、协作协议、健康状态

### 5.5 用户空间 (`resources/users/`)

- **管理内容**: 用户身份、租户信息、用户探索历史
- **记录信息**: 用户账户、角色权限、资源配额、操作审计日志

### 5.6 任务空间 (`resources/tasks/`)

- **管理内容**: 智能处理流程定义及任务实例的运行时状态
- **记录信息**: 流程定义、任务状态（Pending/Running/Done/Failed）、当前步骤、进度、起止时间、关联资源ID列表、执行日志

## 六、核心流程

### 6.1 探索式用户交互流程（简化模式）

```
用户选择数据 → 发出需求
    → 意图解析器分析需求
    → 资源发现器查询工具
        ├─ 有工具 → 调度器分配算力 → 执行器调用工具 → 结果注册到数据空间
        └─ 无工具 → 资源构建器编写代码 → 用户核验 → 注册入库 → 执行
```

### 6.2 多智能体编排流程（复杂模式）

```
用户自然语言描述任务
    → 意图解析器解析步骤/依赖/约束
    → 流程编译器查询 Agent/工具/模型资源
    → 生成编排描述文件 (Markdown)
    → 用户交互修改
    → 确认提交
    → 执行器驱动多Agent协同 → 通信总线 → 状态管理 → 观测反馈
```

### 6.3 AI模型生命周期

```
注册(模型空间) → 关联(工具空间生成调用工具) → 部署(加载到算力节点)
    → 推理(Agent通过调用工具发起请求) → 更新(新版本,旧版本可回溯) → 废弃(归档)
```

## 七、核心设计决策

| # | 决策 | 说明 |
|---|------|------|
| 1 | 探索式交互 | 用户在工作区间选数据、发需求，交互Agent引导全流程 |
| 2 | 工具自动生成 | 无工具时自动编写代码，用户核验后落地为本地工具 |
| 3 | 简化处理模式 | 简单需求确认工具后直接调用执行输出 |
| 4 | 复杂任务编排 | 自然语言驱动自动生成编排描述文件，用户可交互修改 |
| 5 | AI模型双重管理 | 模型空间+工具空间双重建模 |
| 6 | 资源全生命周期 | 所有资源标准化描述、注册登记、核心层统一管理 |

---

## 八、架构优势

- **探索式能力增长**: 用户每次探索产生的工具自动沉淀，工具空间持续增长
- **简化与复杂双模式**: 简单需求直接调用工具快速响应，复杂需求通过多智能体编排完成
- **自然语言驱动**: 用户只需描述需求，系统自动完成工具发现或代码生成
- **人机协同核验**: 自动生成的代码经用户核验方可落地，确保安全性和正确性
- **多智能体协作**: 专业分工的智能体通过通信总线协作
- **全链路可观测**: 监控、告警、追踪、审计四位一体
