# 城市航班与附近飞机查询

## 1. 功能概述

根据用户输入的城市地址（如“北京”、“上海浦东”等），自动解析城市对应的地理坐标，实时获取当前时间该城市机场的航班信息（出发/到达航班号、状态、时间等）以及附近空域中飞机的实时位置（经纬度、高度、速度）。支持自动识别城市主要机场，并可通过缓存降低外部 API 调用频率。

## 2. 输入规范

| 参数名        | 类型   | 必填 | 默认值  | 说明                                                         |
|---------------|--------|------|---------|--------------------------------------------------------------|
| city          | string | 是   | —       | 城市地址，如“北京”、“上海浦东”、“成都”等，支持中/英文名称 |
| radius_km     | number | 否   | 50      | 附近飞机查询范围半径（公里），用于获取周边空域的飞机位置   |
| flight_limit  | int    | 否   | 20      | 返回的航班数量上限（按起飞时间排序）                         |
| airport_code  | string | 否   | "auto"  | 指定机场 IATA 代码，如“PEK”，覆盖自动解析；默认自动识别      |

## 3. 输出规范

| 字段    | 类型   | 说明                                                         |
|---------|--------|--------------------------------------------------------------|
| status  | string | 执行状态 (success / failed)                                  |
| message | string | 结果说明，失败时包含错误详情                                 |
| data    | dict   | 包含以下子字段                                               |
| data.city_info | dict | 城市名称、解析到的坐标、匹配的机场列表                         |
| data.flights | list | 航班信息列表，每项包含：航班号、目的地/出发地、计划/实际时间、状态、机型等 |
| data.aircraft_nearby | list | 附近飞机位置列表，每项包含：呼号、经纬度、高度、速度、航向、最后更新时间 |

## 4. 依赖环境

| 依赖         | 版本    | 用途                       |
|--------------|---------|----------------------------|
| Python       | ≥3.9    | 运行环境                    |
| requests     | ≥2.28   | HTTP 请求调用外部 API       |
| geopy        | ≥2.0    | 城市地址解析至坐标          |
| cachetools   | ≥5.0    | 临时缓存避免重复 API 调用   |

**外部 API（需网络及 API key 或免密钥接口）：**
- **城市坐标解析**：Nominatim (OpenStreetMap) 或预置城市坐标库。
- **航班信息**：AviationStack、FlightAware、Opensky Network 等。
- **附近飞机位置**：OpenSky Network 实时状态接口。

## 5. 运行机制

### 5.1 执行流程

1. 接收城市地址，通过 geopy 或预置库将城市名解析为经纬度坐标（优先从缓存读取）。
2. 根据坐标查找到最近的机场 IATA 代码（可配置加载机场数据库）。
3. 并发调用航班数据 API 获取该机场的出发/到达航班（按当前时间过滤）。
4. 调用飞机追踪 API 获取以城市坐标为中心、指定半径内的实时飞机位置。
5. 将两部分数据整合，按距离或时间排序，过滤无效/过期数据，返回标准输出。

### 5.2 性能指标

- 预期执行时间：< 3 秒（含外部 API 调用，视网络状况）
- 内存占用：< 200MB
- 缓存：城市坐标缓存 1 小时，航班数据缓存 1 分钟（兼顾实时性）

### 5.3 错误处理

- 城市地址无法解析 → 返回 `failed`，提示无法识别城市名称。
- 外部 API 超时或返回 5xx → 重试 1 次，仍失败则返回详细网络错误。
- 查询范围内无飞机或航班数据 → 返回 `success` 但对应列表为空。
- 参数无效（如 radius_km 为负）→ 返回验证错误。

## 6. 测试用例

### 6.1 测试数据描述

```json
{
  "city": "北京",
  "radius_km": 30,
  "flight_limit": 5
}
```

预期输出包含北京首都机场/大兴机场的航班，以及半径30公里内的飞机位置。

```json
{
  "city": "上海浦东",
  "airport_code": "PVG",
  "radius_km": 20
}
```

### 6.2 边界条件

- 输入城市名为空 → 参数校验失败。
- 输入城市名不存在 → 返回城市解析错误。
- radius_km 设置为 1（极小范围）→ 可能返回少量或零架飞机。
- 无网络环境 → 返回网络连接错误。

## 7. 调用示例

```python
from city_flight_tool import execute

result = execute({
    "city": "成都",
    "radius_km": 40,
    "flight_limit": 10
})

print(result["data"]["flights"])
print(result["data"]["aircraft_nearby"])
```
