# 数据清洗工具 - 文本字段提取

## 1. 功能概述

从输入的文本文件中自动提取“颜色”、“沉淀时间”、“评估分数”三个字段，并以 CSV 格式输出结果文件，包含上述三个字段。适用于结构化或半结构化日志、实验记录等文本数据清洗场景。

## 2. 输入规范

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| input_file | string | 是 | - | 待清洗的文本文件路径，支持 .txt、.log 等文本格式 |


## 3. 输出规范

### 3.1 标准输出字段
| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 执行状态 (success/failed) |
| message | string | 结果说明 |
| output_format | string | **必须指定** — file |
| data | dict | 输出数据，格式由 output_format 决定 |

### 3.2 output_format 说明
- `file`: data 含 `{"file_path": "/path/to/output.csv"}` — 文件下载链接或本地路径

## 4. 依赖环境

| 依赖 | 版本 | 用途 |
|------|------|------|
| Python | >=3.8 | 运行环境 |
| pandas | >=1.3.0 | 数据处理与 CSV 生成 |

## 5. 运行机制

### 5.1 执行流程

1. 读取并验证输入参数，检查 input_file 是否存在且可读。
2. 使用指定的分隔符（默认制表符）和编码读取文本文件，尝试将每一行解析为字段列表。
3. 在解析后的数据中定位列名，智能识别包含“颜色”、“沉淀时间”、“评估分数”的列（支持模糊匹配，如“颜色 (nm)”、“沉淀时间(s)”等常见变体）。
4. 若未找到全部三个字段，返回错误并提供已发现的字段列表。
5. 提取匹配的列，按顺序生成包含“颜色”、“沉淀时间”、“评估分数”的 DataFrame，缺失字段填充空值。
6. 将结果写入 CSV 文件至 output_file 路径。
7. 返回状态及输出文件路径。

### 5.2 性能指标

- 预期执行时间: < 5s（1MB 以下文本文件）
- 内存占用: < 500MB

### 5.3 错误处理

- 参数无效（如 input_file 不存在） → 返回 status: failed, message: "输入文件不存在"
- 文本格式无法解析或分隔符不匹配 → 提示检查分隔符和文件结构
- 未找到任何目标字段 → 返回可识别的列名列表，建议调整列名
- 写文件权限不足 → 捕获异常并返回详细错误信息

## 6. 测试用例

### 6.1 测试数据描述

```json
{
  "input_file": "sample_data.txt",
  "output_file": "cleaned_data.csv",
  "delimiter": "\t",
  "encoding": "utf-8"
}
```

示例输入文件内容（制表符分隔）：
```
颜色	沉淀时间	评估分数	备注
红	2.5	8.7	合格
蓝	1.8	9.2	优
绿	3.1	7.8	合格
```

预期输出 CSV 文件内容：
```
颜色,沉淀时间,评估分数
红,2.5,8.7
蓝,1.8,9.2
绿,3.1,7.8
```

### 6.2 边界条件

- 输入文件为空 → 返回空 CSV，仅含表头。
- 目标字段缺失某个：只插入缺失列，其余列用空值填充，仍生成 CSV。
- 列名包含额外字符（如“沉淀时间(s)”）→ 智能匹配后提取。
- 分隔符不是制表符时需正确指定 delimiter 参数。
- 文件编码非 UTF-8 时需指定 encoding 参数，否则可能乱码或解析失败。
