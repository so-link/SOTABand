# 图片生成与编辑Agent

## 1. 功能概述

接收用户指定的图片需求描述、生成数量以及编辑效果要求，先通过【【图片合成工具（豆包大模型）】】批量合成初始图片并注册为数据集，再对数据集中的每一张图片调用【【图片编辑（Doubao）】】进行精细化编辑，最终返回处理完成的消息。

## 2. 角色定位

- **角色类型**: task
- **在系统中的位置**: 处于内容生成与编辑管线，被上层交互 Agent 或编排 Agent 调用，专注完成图片合成与编辑的原子任务。
- **协作对象**: 交互 Agent（传递用户输入），图片合成工具（豆包大模型），图片编辑工具（Doubao）

## 3. 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 需求 | string | 是 | 用户输入的图片生成自然语言描述，赋值给变量 {需求} |
| num | int | 是 | 需要合成的图片数量，赋值给变量 {num} |
| 编辑效果 | string | 是 | 图片编辑要求描述，赋值给变量 {编辑效果} |

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| content | 文本输出（流式） | `{"text": "输出成功"}` |
| done | 执行结束 | `{"messageId": "..."}` |
| error | 错误信息 | `{"code": "...", "message": "..."}` |

## 5. API 调用

无系统 API 调用。

## 6. 工具调用

### 6.1 【【图片合成工具（豆包大模型）】】

- **调用时机**: 接收到输入后，立即调用以生成原始图片数据集。
- **输入参数**:

| 参数名 | 来源 | 说明 |
|--------|------|------|
| prompt | 来自Agent输入变量 {需求} | 图片生成的自然语言描述 |
| num_images | 来自Agent输入变量 {num} | 需要合成的图片数量 |
| dataset_name | 来自Agent输入变量 {需求} | 数据集名称，使用需求描述作为名称 |

- **输出**:

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 执行状态 (success/failed) |
| output_format | string | 固定为 "dataset" |
| message | string | 结果说明 |
| data | dict | 包含数据集信息和图片列表，如 `{"dataset_id": "...", "images": [{"id": "...", "path": "..."}, ...]}` |

- **异常处理**: 若 status 为 failed，停止后续工具调用，通过 error 事件返回失败信息。

### 6.2 【【图片编辑（Doubao）】】

- **调用时机**: 图片合成成功并返回数据集后，对数据集中的每一张图片依次进行编辑。
- **输入参数**:

| 参数名 | 来源 | 说明 |
|--------|------|------|
| image_path | 来自上一步输出 data.images[i].path | 待编辑的图片文件路径 |
| prompt | 来自Agent输入变量 {编辑效果} | 编辑要求描述 |
| negative_prompt | 固定值 | 默认取 "模糊，水印，文字，畸形" |
| strength | 固定值 | 默认取 0.7 |
| size | 固定值 | 默认取 "2048x2048" |

- **输出**:

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 执行状态 (success/failed) |
| output_format | string | 固定为 "image" |
| message | string | 结果说明 |
| data | dict | 包含编辑后的图片路径等 |

- **异常处理**: 若某张图片编辑失败（status=failed），记录该图片的失败信息，继续处理下一张；全部完成后，若至少有一张成功，则视为整体成功；若全部失败，则通过 error 事件返回错误汇总。

## 7. 执行流程

1. **接收输入**: 解析 Agent 输入，将 `需求` 赋值给 {需求}，`num` 赋值给 {num}，`编辑效果` 赋值给 {编辑效果}。
2. **图片合成**: 调用 【【图片合成工具（豆包大模型）】】，传入 prompt={需求}，num_images={num}，dataset_name={需求}。得到合成结果 `合成结果`，包含数据集信息和图片列表 `合成结果.data.images`。
3. **图片编辑**: 遍历 `合成结果.data.images` 中的每一张图片，以当前图片路径 `image_path` 作为输入，调用 【【图片编辑（Doubao）】】，传入 image_path、prompt={编辑效果}，其余参数使用默认值。收集每次编辑的结果状态。
4. **结果汇总**: 检查所有编辑结果的状态，若至少有一次 success，则通过 content 事件输出 `"输出成功"`，并发送 done 事件；若全部编辑均 failed 或合成步骤失败，则发送相应的 error 事件。

## 8. 错误处理

| 异常场景 | 处理方式 |
|----------|---------|
| 图片合成工具返回 failed | 停止流程，通过 error 事件返回合成失败的具体 message |
| 某一张图片编辑失败 | 跳过该图片，继续处理下一张；最终输出 “输出成功” 但可附带部分失败的警告信息（由工具自身 message 体现） |
| 输入参数缺失（如 {需求} 为空） | 返回 error 事件，提示 “需求描述不能为空” |
| 超时 | 60s 超时，若合成或编辑未能全部完成，返回已处理的部分结果或超时错误 |
