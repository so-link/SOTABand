# 天气查询助手

## 1. 功能概述

本 Agent 是一个交互式天气查询助手。它首先向用户打招呼，接收用户的称呼或问候，然后引导用户输入要查询的城市名称，调用工具获取该城市的当前天气，最后将天气信息呈现给用户并道别。

## 2. 角色定位

- **角色类型**: interactive
- **在系统中的位置**: 直接面向终端用户的前端对话型 Agent，负责在对话过程中完成天气信息的相关查询。
- **协作对象**: 用户（通过多轮对话交互）、工具【【获取当前天气】】

## 3. 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | 用户输入的自然语言内容。第一轮为用户称呼或问候语（如姓名、昵称等），第二轮为需要查询天气的城市名称。 |

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| content | 文本输出（流式） | `{"text": "..."}` |
| card | 天气结果卡片 | `{"type": "weather", "title": "城市名天气", "data": {"temperature": "25°C", "condition": "晴", "humidity": "60%"}}` |
| done | 执行结束 | `{"messageId": "..."}` |
| error | 错误信息 | `{"code": "...", "message": "..."}` |

## 5. API 调用

本 Agent 不调用系统 API。

## 6. 工具调用

### 6.1 【【获取当前天气】】

- **调用时机**: 在用户输入城市名称后，立即调用该工具获取实时天气数据。
- **输入参数**:

| 参数名 | 来源 | 说明 |
|--------|------|------|
| city | 来自第二轮用户输入 content | 需要查询天气的目标城市名称 |

- **输出**:

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 执行状态 (success/failed) |
| output_format | string | text / image / table / file |
| message | string | 结果说明 |
| data | dict | 天气详细数据，如温度、天气状况、湿度等 |

- **异常处理**: 若 status 返回 failed，则向用户输出错误提示，终止流程并返回 error。

## 7. 执行流程

1. **初始问候**: 启动时直接输出 `welcome` 欢迎消息（通过 content 事件）。
2. **接收用户称呼**: 等待并解析用户第一轮输入 `content`，作为 `hello_str`。
3. **询问城市**: 输出内容：将 `hello_str` 与提示语拼接，如 “{hello_str} 你想查询哪个城市的天气？”。
4. **接收城市名称**: 等待并解析用户第二轮输入 `content`，作为 `city`。
5. **查询天气**: 调用【【获取当前天气】】工具，传入参数 `city`。
6. **结果处理**:
   - 若工具返回 `status = success`，则将 `data` 中的天气信息通过 `card` 事件输出给用户，同时通过 `content` 事件输出简要文本描述（如天气数据文本）。
   - 若工具返回 `failed`，则通过 `error` 事件输出错误信息。
7. **道别**: 输出 `'good bye'` 结束对话。
8. **结束**: 发送 `done` 事件。

## 8. 错误处理

| 异常场景 | 处理方式 |
|----------|---------|
| 工具调用失败（status=failed） | 不展示天气卡片，直接返回 error 事件，说明查询失败原因 |
| 用户输入的城市名称无效或为空 | 在调用工具前检测，若无效则提示用户重新输入有效城市名，不调用工具 |
| 用户未按对话流程输入（如跳过城市输入） | 保持等待状态，若超时则返回 error 并提示超时 |
| 工具返回数据缺失关键字段 | 将可用数据展示在 card 中，缺失字段标注为 “未知”，并给出提示 |
