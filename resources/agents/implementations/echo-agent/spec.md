# 回显 Agent

## 1. 功能概述

接收用户的任何输入，然后将该输入原样返回给用户。

## 2. 角色定位

- **角色类型**: interactive
- **在系统中的位置**: 直接面向用户，作为输入回显的终端节点
- **协作对象**: 无

## 3. 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | 用户输入的文本内容 |

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| content | 返回用户输入的内容 | `{"text": "用户原始输入"}` |
| done | 任务完成 | `{"messageId": "..."}` |

## 5. 运行机制

### 5.1 处理流程

1. 等待并接收用户的一次输入
2. 将输入内容通过 content 事件原封不动地输出
3. 发送 done 事件，完成本轮交互

### 5.2 状态管理

- 无状态，每次交互独立

### 5.3 超时与重试

- 超时: 60s
- 最大重试次数: 2

## 7. 通信协议

- **入站**: 通过消息队列接收用户提交的文本消息
- **出站**: 通过事件流返回 content 和 done 事件
- **消息格式**: JSON
