# 广州天气定时报告器

## 1. 功能概述

每五秒钟自动获取最新的广州天气信息，并将天气数据以文本形式推送至指定频道或订阅者。

## 2. 角色定位

- **角色类型**: task
- **在系统中的位置**: 作为独立后台任务运行，由调度器进行生命周期管理
- **协作对象**: 天气数据源服务（工具）、消息输出系统（如消息队列或事件总线）

## 3. 输入规范

无外部用户输入。Agent 通过内部定时器触发，无需接收消息或指令（启动/停止可通过系统控制命令管理）。

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| content | 推送天气报告文本 | `{"text": "广州当前天气：晴，温度 28°C..."}` |
| done | 任务主动停止时发出的结束事件 | `{"messageId": "stop-ack"}` |

## 5. 运行机制

### 5.1 处理流程

1. 定时器每 5 秒触发一次执行周期。
2. 调用天气查询工具获取广州实时天气数据。
3. 将天气数据格式化为可读文本。
4. 通过出站通道推送 `content` 事件。
5. 等待下一次触发，循环执行，直到接收到停止信号。

### 5.2 状态管理

- Agent 内部维护 `running` 状态（布尔值）和最后一次执行结果。
- 启动后 `running` 设为 `true`，接收停止命令后设为 `false` 并发出 `done` 事件。

### 5.3 超时与重试

- 单次天气查询超时: 10s
- 查询失败时最大重试次数: 2
- 连续失败超过 5 次时自动暂停并报警

## 7. 通信协议

- **入站**: 通过系统内部命令通道接收启动/停止指令（非周期性数据）
- **出站**: 通过事件总线发布 `content` 和 `done` 事件
- **消息格式**: JSON

## 8. 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| city | "广州" | 目标城市名称 |
| interval | 5 | 查询间隔（秒） |
| max_retry | 2 | 单次查询最大重试次数 |
| fail_threshold | 5 | 连续失败暂停阈值 |
