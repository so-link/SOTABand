# 时间天气汇报员

## 1. 功能概述

该 Agent 定期获取当前系统时间与广州市的天气信息，并按照指定的时间间隔（默认每 10 秒）自动刷新并输出汇报。

## 2. 角色定位

- **角色类型**: task
- **在系统中的位置**: 定时任务执行者，负责周期性数据获取与汇报
- **协作对象**: 天气查询工具（如 OpenWeatherMap 或气象 API）

## 3. 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| - | - | - | 无用户输入，由定时器自动触发 |

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| content | 汇报文本，包含当前时间和天气状况 | `{"text": "当前时间：2025-03-31 14:05:32，广州市天气：多云，28℃"}` |
| done | 汇报结束，进入下一轮等待 | `{"messageId": "..."}` |

## 5. 运行机制

### 5.1 处理流程

1. 定时器触发（每 10 秒）
2. 调用系统时间接口获取当前时间字符串
3. 调用天气查询工具获取广州市天气数据
4. 组合时间与天气信息生成汇报文本
5. 输出 content 事件，然后输出 done 事件
6. 重新进入等待，直至下一次触发

### 5.2 状态管理

- 循环运行中，可通过外部指令暂停或停止
- 暂停状态下不触发定时器，恢复后继续

### 5.3 超时与重试

- 超时: 60s（天气查询单次请求）
- 最大重试次数: 2（仅天气查询失败时重试）

## 6. 工具使用

### 6.1 必选工具

| 工具ID | 工具名称 | 用途 |
|--------|---------|------|
| get_weather | 天气查询 | 获取指定城市的天气信息 |

### 6.2 可选工具

| 工具ID | 工具名称 | 触发条件 |
|--------|---------|---------|
| - | - | - |

## 7. 通信协议

- **入站**: 定时器信号（内部触发）
- **出站**: 向消息通道推送 content 和 done 事件
- **消息格式**: JSON

## 8. 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| city | 广州市 | 目标城市 |
| interval | 10 | 刷新间隔（秒） |
