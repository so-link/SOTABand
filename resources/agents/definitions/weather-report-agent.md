# 广州天气报告 Agent

## 1. 功能概述

定时获取广州市的天气预报，并每隔 10 秒将最新天气信息推送到指定输出通道。

## 2. 角色定位

- **角色类型**: task
- **在系统中的位置**: 独立运行的后台任务 Agent，负责周期性地收集并发布天气数据
- **协作对象**: 天气查询工具、消息分发系统

## 3. 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | 否 | 控制命令：`start` 启动报告，`stop` 停止报告，默认 `start` |

## 4. 输出规范

| 事件类型 | 说明 | 数据结构 |
|----------|------|---------|
| weather_report | 天气报告 | `{"city": "广州", "temperature": "...", "condition": "...", "timestamp": "..."}` |
| status | 状态变更通知 | `{"status": "started"|"stopped"}` |
| done | 任务终止 | `{"message": "task stopped"}` |

## 5. 运行机制

### 5.1 处理流程

1. 接收启动命令后，开始定时循环。
2. 每隔 10 秒调用天气查询工具，获取广州市最新天气。
3. 将天气数据封装为 `weather_report` 事件，发送到消息系统。
4. 若收到停止命令，则退出循环并发送 `done` 事件。

### 5.2 状态管理

- 运行状态：`idle`（初始）、`active`（运行中）、`stopped`（已停止）
- 状态转换由 `action` 输入触发

### 5.3 超时与重试

- 单次天气查询超时: 5s
- 最大重试次数: 2（查询失败后重试间隔1秒）

## 6. 工具使用

### 6.1 必选工具

| 工具ID | 工具名称 | 用途 |
|--------|---------|------|
| weather_query | 天气查询 | 获取指定城市的实时天气信息 |

### 6.2 可选工具

（无）

## 7. 通信协议

- **入站**: 通过消息队列接收控制命令（action）
- **出站**: 将天气报告发布到指定主题
- **消息格式**: JSON

## 8. 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| report_interval | 10 | 报告间隔（秒） |
| city | 广州 | 目标城市 |
| output_topic | weather_reports | 输出消息主题 |
